Precision (Точность)
	Precision - процент правильных положительных прогнозов модели.

				 # positives predicted correctly	   TP
	Precision = --------------------------------- = ---------
				 	# positive predictions			 TP + FP

	Взглянем на модель titanic dataset:
							Действительно верно 	Действительно неверно
	Предсказано верно   	   [233] 					    65
	Предсказано неверно 		109 					   [480]

	Precision = 233 / (233 + 65) = 0.7819
	По сути Precision - мера того, насколько точна модель с её положительными прогнозами.


Recall (Запоминание, отражение)
	Recall - процент положительных случаев, которые модель предсказывает правильно.

			  # positives predicted correctly		TP
	Recall = --------------------------------- = ---------
					# positive cases			  TP + FN

	Recall = 233 / (233 + 109) = 0.6813
	По сути Recall - мера того, сколько положительных случаев может различить модель.


Precision and Recall Trade-off (Компромис)
	Каждый раз при попытке улучшить из этих параметром, другой будет ухудшаться.
	Пример:
		Нужна модель, чтобы предсказать является ли списание с кредитной карты мошенническим. Положительные случаи - мошенническое обвинение, а отрицательные - законные списания.

		Сценарии:
	1. Если мы предположим, что списания мошенническое, мы его отклоним.
	2. Если мы предположим, что списания мошенническое, мы позвоним клиенту, чтобы подтвердить списание.

	В 1-ом случае клиенту неудобно, когда модель срабатывает неверно (ложное срабатывание)
	В 2-ом случае ложное срабатывание доставляет незначительные неудобства покупателю.

	Чем больше ложных срабатываний, тем ниже точность.
	В 1-ом случае было бы полезно иметь низкий Recall и высокую Precision из-за высокой стоимости ложных срабатываний
	Во 2-ом случае нужно больше сбалансировать между Precision и Recall (Уравновесить)

	Нет никаких определённых правил и целей к которым мы стремимся. Всё определяется по ситуации.


F1 Score (Усреднённое между Precision и Recall для сравнения моделей):

				precision * recall
	F1 = 2 * ------------------------
				precision + recall

	Используя предыдущие вычисления:

	F1 = 2 * (0.7819) * (0.6813) / (0.7819 + 0.6813) = 0.7281
	F1 - mean(середина) между Precision и Recall
